  • First is the checkbox input which needs to have a label but could
    also be set to already checked or not
  • Getting it checkedto start with is donw by use of the "Checked" attribute

  • Next up is the radio button.
  • The radio button is like having a checklist from which only one option can
    be selected
  • To connect the options of a radio element, you simply need to make sure that they
    share the same name though id and value may be different.

example radio

  • When submitting a radio, what is sent to the action link is the value of the radio
  • The select input is sort of like a drop down menu
  • it works by combining functionality of the select element and the option element
  • within the "select" opening and closing tags lie the "options" opening and
    closing tags
  • The options tags carry an attribute called value which is what is sent to the form action when
    a user selects an option.

Select input example

  • It's worth noting thast you could have something preselected by keying in the
    "selected" attribute in the options opening tag.

  • Apart from the above is also the Range type of input.
  • It creates a slider which allows users to pick a value on a limited spectrum
  • The actual range of the slider is set using the mmin and max attributes as well as the
    step attribute.
  • The step attribute is particularly important cause it allows progression in
    bits.
  • An initial vqlue can be set usinf the "value" control.

  • A textarea is pretty much what you see in websites asking people to
    submit information
  • You can set the size of the textarea using the "rows" and "cols" attributes in the input
    opening tags.
  • You can also give it a placeholder.
  • Unlike other input methods, you don't have to specify it as an input element and it actually has a closing tag.
  • Neither do you have to put it in a form element.
