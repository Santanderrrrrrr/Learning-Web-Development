divs and spans

  • First remember block elements and inline elements
  • Spans and divs are containers
  • divs are block level content division elements
  • Spans are also generic containers but are inline

reandom elements

  • there's the "hr" tag which has no closing tag and just creates a horizontal line
  • there is also the "br" tag which also has no clocing tag and results in a break between lines
  • there is the "sup" tag which elevates elements from the baseline like powers of numbers and
  • then there's the "sub" tag that drops them under like in logariths
  • sup and sub have closing tags and you wrap elements in them

Semantic elements

  • There are certain things known as "semantic elements" that can be used instead of divs:
    1. section
    2. article
    3. nav
    4. main
    5. header
    6. footer
    7. aside
    8. summary
    9. details
  • These make your code a little bit more elegant
  • Main is the dominant content of the body of a page and
    excludes content that could be repeated across the body of a page
  • Nav represents anything on the page that provides navigation
  • A sction is a standalone segment that doesn't have a more specific semantic element
  • the article element contains a composition in a document
  • aside is tangential and not essential: think callout boxes, side bars
  • header is introductory and may include the nav while the footer usually just has navs
  • neither of these definitions are definitive.
    all of these elements are pretty flexible. The labelling is just to make cooperation
    easy
  • finally there's the time element which is used to encapsulate
    time and format it in a manner the machine could read and not necessarily just string
